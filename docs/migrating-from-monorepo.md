# Migrating from a monorepo

This guide maps domain-specific Bazel primitives historically used in
a monorepo onto their domain-agnostic counterparts in
`rules_coen_applied_lifecycle`.

## Symbol mapping

| Old symbol / pattern | New symbol / pattern | Notes |
|---|---|---|
| `k8s_service_manifest` | `kustomize_manifest` | Now a real rule, not a macro globbing `k8s/**`. Callers pass `base_manifests`, `overlay_manifests`, and `kustomization` explicitly. |
| `k8s_manifests_oci` | `manifests_oci_push` (default `layout = {"*": "manifests"}`) | Default layout preserves `/manifests/<pkg>/…` archive paths. |
| `k8s_service_manifests_oci` | `manifests_oci_push(flat = True)` | `flat = True` == basename-deduped layout for single-service archives. |
| `service_oci_push` | `application_oci_push` | Identical semantics; renamed to drop the "service" noun. |
| `select_env(dev=…, staging=…, prod=…)` | `select_env({"dev": …, "staging": …, "prod": …}, flag_package = "//your/flags/pkg")` | Mapping is now a dict keyed on caller-supplied env names. An explicit `flag_package` is required — no hardcoded `//build_flags` default. No silent fallback to `dev` via `//conditions:default`; pass `default=` if you want one. |
| `//build_flags:pipeline` | `//<your-flags-pkg>:environment` (generated by `coen_applied_lifecycle_environments`) | The flag label lives wherever you invoke `coen_applied_lifecycle_environments(...)`; its name is always `environment`. |
| Runtime `bazel query 'attr(name, "-manifests-push$", ...)'` | `coen_applied_release_group(name, pushes = [...])` | Aggregation is an analysis-time graph, not a runtime shell-out. |
| Hand-rolled `rewrite_images.py` invocations | `image_refs = {"src": "dst", ...}` attribute on `kustomize_manifest` / `helm_manifest` | Bare repo DSTs are auto-qualified using the `tag_flag` and `registry_flag` providers. |
| `_manifests_tar` / `_service_manifests_tar` rules | `manifests_oci_layout` (one rule; `flat = True` or an explicit `layout`) | One rule covers both fleet and per-service use cases. |

## Drop-in steps

1. In your consuming workspace, add the `bazel_dep` + archive_override:

   ```starlark
   bazel_dep(name = "rules_coen_applied_lifecycle", version = "0.1.0")
   archive_override(
       module_name = "rules_coen_applied_lifecycle",
       urls = ["<release artifact url>"],
       strip_prefix = "rules_coen_applied_lifecycle-0.1.0",
       integrity = "sha256-<artifact sha256>",
   )
   ```

2. Replace your old `build_flags/flags.bzl` file with a single call in a
   dedicated package (e.g. `deploy/flags/BUILD.bazel`):

   ```starlark
   load(
       "@rules_coen_applied_lifecycle//lifecycle:defs.bzl",
       "coen_applied_lifecycle_environments",
   )

   coen_applied_lifecycle_environments(
       environments = ["dev", "staging", "prod"],  # or whatever your set is
       default_environment = "dev",
       default_image_registry = "ghcr.io/yourorg",
   )
   ```

3. Search-and-replace the symbol mappings above across all service BUILD
   files. Every call now needs three keyword-required flag labels:
   `env_flag`, `tag_flag`, `registry_flag`. Wire each to the targets
   generated in step 2.

4. Replace `//k8s:manifests_push_all.sh` (or whatever your release runner
   is) with a single `coen_applied_release_group` target. Have your release
   pipeline `bazel run //path/to:release_group` instead of running
   `bazel query | xargs bazel run`.

5. Remove the copy of `rewrite_images.py` living in
   `build_defs/k8s/`. The ruleset ships its own hermetic rewriter.

6. Confirm `bazel test //...` passes in your workspace (whatever tests you
   already have). If you additionally want per-env rendering assertions,
   copy the `diff_test` patterns from `tests/tests/BUILD.bazel` in
   this repo.

## Common pitfalls

- **Forgetting `flag_package`** on `select_env` — now mandatory. The old
  helper implicitly used `//build_flags`; the new one refuses to guess.
- **Passing a bare source file to `manifests_oci_layout`'s `manifests`
  attribute** — wrap them in a `filegroup` so the attribute's
  `providers = [[LifecycleManifestsInfo], [DefaultInfo]]` constraint is
  satisfied.
- **Relying on `select_env({"dev": …}, default = X)` defaulting to `X`
  when the env flag is `prod`** — not how this version works. The new
  helper emits `//conditions:default` only if you explicitly pass
  `default = …`. Selecting an env that's not in the mapping is a hard
  failure.

## Overriding pinned toolchains

The ruleset registers pinned `kustomize`, `helm`, and `rules_python`
toolchains on consumers' behalf (see `MODULE.bazel` for exact versions).
A consumer that needs a different version can register their own
toolchain earlier in their `MODULE.bazel`; Bazel's toolchain resolution
picks the first matching registration.
